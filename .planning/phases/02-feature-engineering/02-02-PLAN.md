```python
class SpacingMetrics:
    game_id: str

    frame_number: int

class DefensivePressure:
    game_id: str
    track_id: int
    frame_number: int
    nearest_defender_distance: float   # pixels to nearest opposing player
    closing_speed: float               # px/frame change; negative = closing
    timestamp_ms: float
```

<feature>
  <name>Player Spacing Metrics (FE-01)</name>
  <files>features/spacing.py, tests/test_spacing.py</files>
  <behavior>
    - Square of side 10: 4 corners → convex_hull_area = 100.0
    - Two points only (fewer than 3) → convex_hull_area = 0.0, avg_inter_player_distance = distance between the two
    - Single point → both metrics = 0.0
    - Empty list → both metrics = 0.0
    - Five players in a known configuration → avg_inter_player_distance matches hand-computed mean pairwise distance (within 1e-6 tolerance)
    - Returns SpacingMetrics dataclass with correct game_id, frame_number, timestamp_ms passed through
  </behavior>
  <implementation>
    features/spacing.py:
    - Import scipy.spatial.ConvexHull for hull area (use scipy not shapely — scipy is in requirements)
    - compute_spacing(player_positions: list[tuple[float, float]], game_id: str, possession_id: str, frame_number: int, timestamp_ms: float) -> SpacingMetrics
    - Hull area: ConvexHull(points).volume for 2D (ConvexHull.volume IS the area in 2D in scipy)
    - Fall back to 0.0 if fewer than 3 points or scipy raises QhullError (collinear points)
    - Avg inter-player distance: itertools.combinations(positions, 2) → mean of math.hypot distances
    - Return SpacingMetrics(...) with computed values
  </implementation>
</feature>

<feature>
  <name>Defensive Pressure Metrics (FE-02)</name>
  <files>features/defensive_pressure.py, tests/test_defensive_pressure.py</files>
  <behavior>
    - Offensive player at (0,0), defenders at (3,4) and (10,0) → nearest_defender_distance = 5.0
    - closing_speed when previous nearest distance was 8.0 and current is 5.0 → closing_speed = -3.0 (closing)
    - closing_speed when previous nearest distance was 3.0 and current is 5.0 → closing_speed = 2.0 (opening)
    - closing_speed = 0.0 when no previous distance available (first frame for this track_id)
    - Empty defender list → nearest_defender_distance = float('inf'), closing_speed = 0.0
    - Returns one DefensivePressure per offensive player track_id
    - prev_distances dict is mutated in place with new values after computation
  </behavior>
  <implementation>
    features/defensive_pressure.py:
    - compute_defensive_pressure(player_rows: list[dict], game_id: str, frame_number: int, timestamp_ms: float, prev_distances: dict[int, float]) -> list[DefensivePressure]
    - player_rows: each dict has keys track_id (int), x (float), y (float), team (str or None)
    - Split players into offensive/defensive by team field; if team is None, treat all as potential pressure targets and use all others as defenders
    - For each player, compute min Euclidean distance to all other players using math.hypot
    - closing_speed = current_dist - prev_distances.get(track_id, current_dist) — first frame gives 0.0
    - Update prev_distances[track_id] = current_dist after computing
    - Use only math and dataclasses — no numpy required for this module
  </implementation>
</feature>
