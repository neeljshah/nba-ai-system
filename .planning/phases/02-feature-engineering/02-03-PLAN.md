<feature>
  <name>Off-Ball Event Detection (FE-03)</name>
  <files>features/off_ball_events.py, tests/test_off_ball_events.py</files>
  <behavior>
    - Cut: player speed > CUT_SPEED_THRESHOLD (default 200 px/s) AND moving toward basket area (direction toward x>400 half of court) → event_type='cut', confidence proportional to speed
    - Screen: player speed < SCREEN_SPEED_THRESHOLD (default 30 px/s) AND another player is within SCREEN_PROXIMITY (default 60 px) and moving faster → event_type='screen', confidence=0.8
    - Drift: player speed in 30–100 px/s range AND moving laterally away from ball (cross product of ball-player vector and velocity vector indicates away motion) → event_type='drift', confidence=0.6
    - A player triggers at most one event per frame (priority: cut > screen > drift)
    - Empty frame_sequence → returns []
    - Player with no detectable movement pattern → no event emitted for that player
    - detect_off_ball_events is called per-frame (frame_sequence is a sliding window of recent frames; ball_pos is current frame's ball position)
  </behavior>
  <implementation>
    features/off_ball_events.py:
    - Constants at top: CUT_SPEED_THRESHOLD = 200.0, SCREEN_SPEED_THRESHOLD = 30.0, SCREEN_PROXIMITY = 60.0, DRIFT_SPEED_MIN = 30.0, DRIFT_SPEED_MAX = 100.0
    - detect_off_ball_events(frame_sequence: list[list[dict]], game_id: str, ball_pos: dict | None) -> list[OffBallEvent]
    - Use only the most recent frame in the sequence for per-player classification (frame_sequence[-1])
    - Screen detection: O(n²) proximity check among players in current frame
    - All math via standard math module — no numpy needed
    - basket_x = 470.0 (right basket in standard pixel space; hardcoded constant; court-calibration deferred to future phase)
  </implementation>
</feature>

<feature>
  <name>Pick-and-Roll Detection (FE-04)</name>
  <files>features/pick_and_roll.py, tests/test_pick_and_roll.py</files>
  <behavior>
    - PnR condition: in frame window, one player (screener) is slow (speed < 40 px/s) AND within SCREEN_DISTANCE (80 px) of another faster player (ball handler, speed > 80 px/s), AND then the ball handler moves away (distance increases in next frames)
    - Returns PickAndRollEvent with ball_handler_track_id and screener_track_id (ball handler = faster player)
    - Two PnR events in the same window with same pair → deduplicate, emit once per pair per window
    - frame_sequence must have >= PNR_WINDOW_FRAMES (default 5) frames to detect — shorter → returns []
    - If multiple screener candidates exist, pick the one closest to ball handler
  </behavior>
  <implementation>
    features/pick_and_roll.py:
    - Constants: PNR_WINDOW_FRAMES = 5, SCREEN_DISTANCE = 80.0, SCREENER_MAX_SPEED = 40.0, HANDLER_MIN_SPEED = 80.0
    - detect_pick_and_roll(frame_sequence: list[list[dict]], game_id: str) -> list[PickAndRollEvent]
    - Analyze the window: for each pair (fast_player, slow_player), check if within SCREEN_DISTANCE in the middle frame of the window
    - Verify the separation: in the last frame, distance should be > SCREEN_DISTANCE (handler moved off screen)
    - Use a set of (ball_handler_track_id, screener_track_id) tuples to deduplicate within a window
    - frame_number and timestamp_ms taken from the middle frame of the window
  </implementation>
</feature>
