  <action>
    Create models/player_impact.py implementing ML-04.

    EPA formula context: Expected Points Added per 100 possessions.
    Base rate: NBA average ~107 points per 100 possessions.
    Player adjustment: model predicts deviation from league average
    based on shot quality (court_zone proxy via x,y) and event contribution.

    Feature engineering inside fit():
    Per track_id, aggregate from player_event_data:
    - made_rate: mean(made)
    - shots_taken: count of rows
    - cut_rate: fraction of events where event_type='cut'
    - screen_rate: fraction where event_type='screen'
    - avg_confidence: mean(confidence)
    - x_mean: mean(x)  — court zone proxy (closer to basket = higher value)

    Label (y — EPA deviation):
    y = (made_rate - 0.45) * 2.2 * 100  # deviation from avg efficiency scaled to per-100
    This is a regression label, not classification.

    Estimator: RandomForestRegressor(n_estimators=100, max_depth=4, random_state=42)

    If df is empty or fewer than 5 unique track_ids: generate synthetic data:
    10 synthetic players with made_rate~U(0.30, 0.65), shots_taken~randint(5,20),
    cut_rate~U(0,0.5), screen_rate~U(0,0.3), avg_confidence~U(0.6,0.95),
    x_mean~U(100,800). Compute label y from formula above.

    predict(features: dict) -> dict:
    - features keys: track_id (int, for logging only), made_rate (float),
      event_mix (dict with event_type counts), shots_taken (int)
    - Derive: cut_rate = event_mix.get('cut',0) / max(shots_taken,1)
               screen_rate = event_mix.get('screen',0) / max(shots_taken,1)
               avg_confidence = 0.75  (no confidence at predict time; use mean)
               x_mean = 400.0  (neutral court position at predict time)
    - Return {'epa_per_100': float(clf.predict([[made_rate, shots_taken, cut_rate, screen_rate, 0.75, 400.0]])[0])}
