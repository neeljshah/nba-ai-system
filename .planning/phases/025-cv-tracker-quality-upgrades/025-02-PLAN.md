# Plan 025-02: Jersey OCR Brightness Normalisation + 2x Resize

## Goal
Fix 0 jersey names in 29K tracking rows by improving OCR preprocessing:
- Add brightness normalisation (histogram stretch) after CLAHE
- Add a 3rd OCR pass on a 2x-upscaled crop (small broadcast crops are the primary failure)

## Context (from code audit)

Current `jersey_ocr.py`:
- `preprocess_crop()` already does: slice 20-70% rows → upscale to 64px → CLAHE → adaptive threshold
- `read_jersey_number()` already does: normal pass + inverted pass → picks best conf ≥ 0.65
- `_OCR_CONF_MIN = 0.65` already exists (no change needed — already > 0.6)
- **Missing**: histogram stretch after CLAHE in preprocess_crop
- **Missing**: 3rd OCR pass on 2x-resized image in read_jersey_number

## Tasks

<task id="B1" file="src/tracking/jersey_ocr.py">
In `preprocess_crop()`, add brightness normalisation after the CLAHE step:

Replace the existing gray/clahe/binary block:
```python
# Grayscale + CLAHE for local contrast
gray = cv2.cvtColor(roi, cv2.COLOR_BGR2GRAY)
clahe = cv2.createCLAHE(clipLimit=3.0, tileGridSize=(4, 4))
enhanced = clahe.apply(gray)
```

With:
```python
# Grayscale + CLAHE for local contrast
gray = cv2.cvtColor(roi, cv2.COLOR_BGR2GRAY)
clahe = cv2.createCLAHE(clipLimit=3.0, tileGridSize=(4, 4))
enhanced = clahe.apply(gray)

# Brightness normalisation — histogram stretch to full 0-255 range
e_min, e_max = int(enhanced.min()), int(enhanced.max())
if e_max > e_min:
    enhanced = cv2.normalize(enhanced, None, 0, 255, cv2.NORM_MINMAX)
```

Keep the adaptive threshold block unchanged after this.
</task>

preprocessed = preprocess_crop(crop_bgr)

# Pick best result across all three passes
best_number: Optional[int] = None
best_conf: float = -1.0

for results in (results_normal, results_inverted, results_2x):

</task>
