<task type="auto" tdd="true">
  <name>Task 2: Court line and zone boundary detector</name>
  <files>pipelines/court_detector.py</files>
  <behavior>
    - CourtDetector.detect_lines(frame: np.ndarray) returns List[CourtLine] where CourtLine is a dataclass with: rho (float), theta (float), x1 (int), y1 (int), x2 (int), y2 (int)
    - CourtDetector.detect_zones(frame: np.ndarray) returns CourtZones dataclass with: paint_region (list of 4 corner points or None), three_point_arc_points (list of points or None), half_court_line (tuple of two points or None)
    - detect_lines uses: grayscale conversion, Gaussian blur (kernel 5x5), Canny edge detection (low=50, high=150), HoughLinesP (threshold=80, minLineLength=100, maxLineGap=10)
    - detect_zones uses color-based masking (HSV court floor color range) + contour detection to approximate zone boundaries — returns None for zones it cannot detect rather than crashing
    - CourtDetector is designed to be called once per game (or periodically) to calibrate spatial reference — not necessarily every frame
    - Both methods handle empty/failed detection gracefully (return empty list or None fields)
  </behavior>
  <action>
    Create pipelines/court_detector.py:
      - Import cv2, numpy, dataclasses
      - Define CourtLine dataclass: rho: float, theta: float, x1: int, y1: int, x2: int, y2: int
      - Define CourtZones dataclass: paint_region: list | None, three_point_arc_points: list | None, half_court_line: tuple | None
      - CourtDetector class (stateless, no __init__ needed beyond optional config)
      - detect_lines(frame): gray = cv2.cvtColor, blur = cv2.GaussianBlur((5,5),0), edges = cv2.Canny, lines = cv2.HoughLinesP. Convert each line [x1,y1,x2,y2] to CourtLine (compute rho/theta from endpoints for consistency). Return [] if lines is None.
      - detect_zones(frame): apply HSV mask for hardwood floor color (hue ~15-30, high saturation/value). Find contours. Attempt to identify paint rectangle (largest rectangular contour in lower third of frame), half-court line (horizontal line near vertical center), three-point arc (curved contour near paint). Return CourtZones with None for undetected elements.

    NOTE: NBA broadcast court detection using classical CV is heuristic and imperfect. This implementation provides a best-effort spatial reference. Phase 1 goal is that the output is "usable as spatial reference" — not pixel-perfect. Do not over-engineer this; the system must work for Phase 2 feature engineering.

    Color ranges for hardwood detection (NBA court):
      - HSV lower: (10, 50, 100)
      - HSV upper: (30, 255, 255)
    These are starting values; comment that they may need tuning per video source.
